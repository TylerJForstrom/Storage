How to read this

  • WHERE id = … descends the B+tree: root → leaf, a handful of page reads no matter how big the table gets.
  • WHERE age = … can't use the key, so the engine walks every leaf — watch the whole grid light up.
  • Writes go to the write-ahead log first (purple), get fsynced, and only then into the database pages — that ordering is why a crash can't corrupt committed data.
  • Run EXPLAIN on anything to see the planner's reasoning without executing.

Built from scratch by Tyler Forstrom — Rust core with zero dependencies and no unsafe code, differential-fuzzed against BTreeMap and crash-tested at every disk operation.
